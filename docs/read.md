案例研究：GLOBAL LUXURY CRUISE LINES（GLCL）
Global Luxury Cruise Lines（GLCL）是一家高级豪华邮轮公司，负责为多个国际邮轮运营商管理全球预订业务。其总部位于马来西亚吉隆坡。
乘客可以预订以下四种舱房类别之一：
•	Interior 内舱
•	Ocean View 海景舱
•	Balcony 阳台舱
•	Suite 套房
邮轮航线类型包括：
•	One-way 单程航线，例如调船航线
•	Round-trip 往返航线
•	Multi-destination 多目的地航线，例如跳岛行程
根据不同船只，邮轮提供多种餐饮选择，包括：
•	标准固定时间用餐
•	灵活任意时间用餐
•	特色餐厅
航程长度会决定乘客获得的是标准登船餐，还是多日全包餐饮套餐。
根据年龄，邮轮也会提供特殊服务，例如：
•	船上儿童托管服务
•	青少年专属俱乐部
票价会根据以下乘客年龄类别计算：
•	Infant 婴儿：2岁以下
•	Child 儿童：2至12岁
•	Teen 青少年：13至17岁
•	Adult 成人：18至59岁
•	Senior 老年人：60岁或以上
婴儿票价取决于他们是否需要独立婴儿床，或者是否与他人共用现有床位。
如果婴儿共用床位，邮轮会收取成人票价的 15% 作为象征性费用。
如果婴儿需要婴儿床或小床，则收取儿童票价的 50%。
每个预订中的舱房最多只能容纳 5名乘客。
根据安全规定，17岁或以下的青少年不能单独入住一个舱房，除非有18岁或以上的成人监护人被安排在相邻或连接的舱房中。
另外，部分邮轮运营商为15至17岁的教育旅行青少年提供 “Chaperoned Youth” 陪同青少年计划，但需要支付强制性的每日监督费用。
所有邮轮运营商都会根据需求提供：
•	轮椅无障碍舱房
•	行动辅助服务
所有预订都必须遵守一般限制和海事规则，包括严格的行李重量限制。
乘客可以取消预订或更改行程，但可能需要支付额外费用，并且必须遵守票价规则。
如果邮轮尚未启航，船票可以转移到新的日期，但新的航程必须在原始预订日期起 一年内 开始。
如果乘客在邮轮从母港计划出发前 48小时内 取消或尝试更改行程，则会损失整张船票的价值。
 
项目范围
你们需要设计一个数据库，涵盖以下领域：
•	邮轮预订
•	邮轮取消与重新安排
 
作业要求
1. 设计
1(a) ERD 与逻辑设计
开发一个 ERD，用于支持 GLCL 的业务活动。
ERD 必须显示：
•	实体
•	实体之间的关系
•	主键 Primary Key
•	外键 Foreign Key
•	基数 Cardinality
•	可选性 Optionality
ERD 后面还需要附上 logical design。
数据必须符合 3NF 或更高范式。
如果因为性能原因进行了反规范化 denormalization，则必须提供详细解释。
你的数据模型应完整支持案例研究中描述的所有业务需求。
分数：30分
 
1(b) 优化策略
记录并提供一份关于优化策略的书面描述和说明。
每位组员都需要提出 一个优化策略。
分数：10分

每位组员都需要设计 一个 constraint 和一个 trigger。
分数：10分
